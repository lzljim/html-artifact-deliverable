# Stage 6: 整理、归档和备份

## 问题与目标

Stage 6 面向个人数据生命周期：归档表示默认不看，常用资料表示会反复查看，完成表示内容状态。目标是让整理视图能发现语义冲突，让导入导出更适合换机器和恢复备份。

## 验收标准

- 整理视图继续保留长期未打开、已完成未归档、无项目集、无阶段。
- 整理视图新增“归档但仍是资料”和“归档但仍置顶”，支持批量移出资料和取消置顶。
- 导入全量 bundle 前先预览新增、冲突和项目集数量。
- 冲突导入需要明确确认，确认后才覆盖同名 artifact。
- JSON 文件存储和 no-build frontend 不变。

## 不做范围

- 不引入 SQLite 或独立备份仓库。
- 不做自动清理或自动归档。
- 不删除 artifact 文件。
- 不改变 `done`、`reference`、`archived` 的语义。

## Review Slices

1. `done` 整理视图增加归档/资料/置顶语义冲突分组。
2. `done` 新增导入预览 API 和首页导入前确认。
3. `done` 更新测试、文档和 roadmap checkpoint。
4. `done` 本地检查、浏览器验证、提交并推送。

## 验证计划

- `npm run check`
- 浏览器验证整理视图新增分组。
- 浏览器验证导入 JSON 时先显示预览和确认。
